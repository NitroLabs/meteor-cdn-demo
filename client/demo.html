Meteor CDN Demo

The static resources on this page are served from {{CDN_URL}}



Faster page loads

Static resources are served from a different domain increasing download parallelization

Reduced bandwidth

Serving files from a CDN reduces the bandwidth consumed by the application servers.

Better Caching

Using a CDN allows almost all of your application code to be cached.

Proper 404s

By default Meteor returns a 200 reponse code, even for missing resources. CDN ensures that Meteor returns a proper 404 missing resources. Try loading missing.css or expired.js

Cross Origin Headers

Meteor CDN adds cross-origin and STS headers to webfonts to solve cross-origin problems in Google Chrome

Template Helper

The template helper {{CDN_URL}} provides a quick way to get the CDN url in Blaze templates

Meteor CDN was developed by Nitrolabs for the Meteor Community. The source for this demo is available on Github. The Meteor CDN package can be found on Github and Atmosphere. Suggestions and pull requests are welcome.
